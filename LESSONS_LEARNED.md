### 錯咩
計PUT回報時，將Strike價格直接當成打和點，冇減去期權金成本。

### 例子
```
錯：$55K PUT，BTC到$55,000就賺錢
對：$55K PUT成本$731，Breakeven = $55,000 - $731 = $54,269
    BTC到$55,000係ATM，內在值=$0，輸清$731

### 防錯機制
- ✅ Breakeven公式：PUT = Strike - Premium/張；CALL = Strike + Premium/張
- ✅ 每次計算回報表時，第一行就標Breakeven

```

## 🚨 錯誤10：忽略Gamma效應 — 只用固定Delta或到期結算（致命！💀💀💀）

### 錯咩
計算回報時，只用兩種極端：
1. 到期結算（等14日TWAP）
2. 固定Delta短線（假設Delta唔變）

但**真正嘅波動交易，Delta係會變嘅（Gamma）。** BTC越接近Strike，Delta越大，張期權升得越快。呢個先係「接近個價張期權都會升」嘅真正含義。

### 例子
```
錯（固定Delta計法）：
    $62K CALL，Delta 0.37，BTC升$1,000 → CALL升$370 → 要升到$62,963先打和

真相（Delta + Gamma路徑計法）：
    BTC由$60K升到$62K：
    $60K: Delta 0.37, Gamma 0.0003
    $60.5K: Delta 0.42 → 呢段升咗$198
    $61K: Delta 0.48 → 再升$225
    $61.5K: Delta 0.55 → 再升$258
    $62K: Delta 0.62 → 再升$293
    累計升幅 ≈ $974！唔使等到$63K打和！

即係BTC只升3.3%到$62K，$62K CALL已經賺咗超過$800！
而唔係我之前計嘅「升到$62K仲係OTM歸零」。
```

### Gamma效應關鍵數字（以 $62K CALL 為例）
| BTC由$60K升到 | Delta變化 | 每段CALL升幅 | 累計CALL升幅 |
|-------------|----------|------------|------------|
| $60.5K | 0.37→0.42 | +$198 | +$198 |
| $61.0K | 0.42→0.48 | +$225 | +$423 |
| $61.5K | 0.48→0.55 | +$258 | +$681 |
| **$62.0K** | **0.55→0.62** | **+$293** | **+$974 🟢** |

> BTC升3.3%，CALL升咗$974（超過10倍槓桿！）
> 呢個先係期權真正嘅威力 — Gamma加速效應

### 防錯機制
- ✅ 計算期權回報時，必須考慮Gamma路徑，唔好只用固定Delta
- ✅ 越接近Strike，Delta越大，回報加速
- ✅ **期權最大嘅優勢唔係「平」，係「非線性加速」**
- ✅ 揀Strike時，揀「Gamma最大」（即最接近現價嘅OTM），唔好揀「最平溢價」

## 🚨 錯誤8：冇先問「點解唔用期貨？」（框架性錯誤💀💀💀）

### 錯咩

### 例子
```
同樣睇淡，目標$55K：
    期貨做空0.01 BTC → BTC跌到$55K賺$5,000
    $55K PUT        → BTC跌到$55K蝕清$731
                     → 要跌到$54,269先打和

```

### 防錯機制
- ✅ 推期權前，必須先答：「呢個方向，點解唔用期貨？」
- ✅ 比較期貨線性回報 vs 期權@目標回報
- ✅ 如果期權冇明顯優勢（槓桿/非線性/時間值），推期貨
- ✅ 0.01張規模：期貨幾乎永遠贏
