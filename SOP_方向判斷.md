# 🎯 方向判斷SOP — 先Scan方向，後Scan期權

> **教訓：唔好同時掃CALL+PUT。先判斷方向，再掃嗰一邊嘅期權。**
> 生成日期：2026-06-26

---

## 流程

```
Step 1: 方向判斷（開3-5個subagents同時驗證）
├─ Gann正方：現價 vs 關鍵線，方向偏上/下？
├─ Fib時間節點：而家喺邊個時間位？轉折未？
├─ Elliott Wave：而家咩浪？確認率幾多%？
├─ 鏈上數據：邊個接貨？邊個沽貨？大戶方向？
├─ 爆倉位地圖：邊個方向更易被逼倉？
├─ 資金費率：空頭crowded定多頭crowded？
└─ Order Book結構：近端Bid/Ask邊個薄？結算日效應？

Step 2: 方向確認 → 評分
├─ CALL > 60% → 只掃CALL
├─ PUT > 60% → 只掃PUT
└─ 50/50 → 雙開Strangle或等

Step 3: 根據方向掃Derive.xyz期權
├─ DTE同目標對齊
├─ 打和點合理（低過目標價）
├─ Spread/流動性合格
└─ 成本可控

Step 4: 入場
├─ 掛Limit，唔好市價追
├─ 分注食糊（對齊爆倉位）
└─ 止蝕鐵律
```

### 點解結算前會有巨震？

| 機制 | 解釋 | 影響 |
|------|------|------|
| **Dealer Gamma對沖** | 做市商持大量期權，越近結算Gamma越急 → 對沖頻率越高 → 價格波動放大 | 震盪加劇 |
| **Max Pain磁石效應** | 莊家傾向將結算價推向最多期權歸零嘅位，減少賠付 | 人為壓價/抬價 |
| **流動性蒸發** | 結算前做市商撤Bid/Ask（Call Wall Bid=0），Order Book變薄 | 細單推得郁價 |
| **多空最後廝殺** | 結算前最後機會逼倉對方 — 多頭推高、空頭壓低 | 雙向爆倉 |
| **Gamma解鎖釋放** | 結算完後Gamma枷鎖解除，價格自由行 | 出真方向 |
