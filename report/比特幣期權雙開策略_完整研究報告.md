## 一、什麼是比特幣期權雙開（Straddle / Strangle）

### 1.1 跨式策略（Straddle）

**Long Straddle（買入跨式）** — 同時買入相同行使價（Strike Price）、相同到期日的 Call（看漲期權）和 Put（看跌期權）。

這是一種**非方向性策略**（Delta Neutral）：交易者不在乎價格漲跌，只在乎價格**波動幅度是否足夠大**。

**損益結構：**
| 指標 | 數值 |
|------|------|
| 最大虧損 | 支付的兩邊權利金總和（Call權利金 + Put權利金） |
| 上方損益平衡點 | 行使價 + 總權利金 |
| 下方損益平衡點 | 行使價 - 總權利金 |
| 潛在收益（上漲） | 理論上無限 |
| 潛在收益（下跌） | 理論上至價格歸零 |

**Short Straddle（賣出跨式）** — 同時賣出相同行使價、相同到期日的 Call 和 Put。收取權利金，賭價格不會大幅波動。最大收益為收取的總權利金，但潛在虧損理論上無限。

### 1.2 勒式策略（Strangle）

> ⚠️ **先搞清楚**：Straddle 同 Strangle 都係**非方向性策略（Delta Neutral）**。你唔需要知升定跌，只賭**波動幅度**夠唔夠大。呢個同方向性策略（淨買Call睇升 / 淨買Put睇跌 / 價差等）係完全兩回事。

**Long Strangle（買入勒式）** — 買入行使價不同的 Call 和 Put：
- Call 行使價高於現價（OTM）→ 要升到呢個位先開始ITM
- Put 行使價低於現價（OTM）→ 要跌到呢個位先開始ITM

成本比 Straddle **低好多**（因為兩邊都係OTM，期權金平），但OTM行使價創造咗一個**「死亡區間」**：價格如果留喺兩個行使價之間冇爆出去，兩邊到期都OTM變廢紙，**全蝕**。

> Strangle 嘅 trade-off 唔係「需要更大波動」，而係：**有個全蝕區間，但只要突破死亡區間，因為成本低，Breakeven可以比Straddle更窄，回報率更高。**

**Short Strangle（賣出勒式）** — 同時賣出價外的 Call 和 Put，收取權利金，賭價格保持在兩個行使價之間的範圍內。

#### Straddle vs Strangle 實例對比（BTC = $60,000，7/31到期，IV ~48%）

| | Long Straddle（ATM） | Long Strangle（5% OTM） |
|------|:---:|:---:|
| **Call行使價** | $60,000 | $63,000（+5%） |
| **Put行使價** | $60,000 | $57,000（-5%） |
| **單邊權利金（約）** | ~$3,800 | ~$1,500 |
| **總成本** | ~$7,600 | ~$3,000 |
| **上方Breakeven** | $67,600 | **$66,000** |
| **下方Breakeven** | $52,400 | **$54,000** |
| **需要升幅先賺** | +12.7% | **+10.0%** |
| **需要跌幅先賺** | -12.7% | **-10.0%** |
| **最大虧損** | $7,600（100%） | $3,000（100%） |
| **成本比率** | 100% | **39%** |

### 1.5 雙開分多空 — Long vs Short 全面對比

雙開策略有**兩種完全相反的方向**，必須搞清楚先落場：

| | 🔴 Long（買方 / 多波動） | 🟢 Short（賣方 / 空波動） |
|------|------|------|
| **操作** | 買入 Call + 買入 Put | 賣出 Call + 賣出 Put |
| **賭咩** | 價格會**大幅波動**（升跌都得） | 價格**唔會大波動**（窄幅上落） |
| **最大利潤** | 理論上無限（升跌都賺） | 有限（收取的總權利金） |
| **最大虧損** | 有限（蝕晒付出的總權利金，100%） | 理論上無限（爆邊輸突） |
| **風險回報比** | 有限蝕（100%期權金）、無限賺 | 有限賺（期權金）、無限蝕（爆邊輸突） |

> **⚠️ 關鍵：Short 方最大虧損 ≠ 期權金！** 好多人搞錯以為 Short Straddle 最多輸晒收嘅期權金，**完全錯**。實例：BTC=$60K，Short ATM Straddle 收$4K。若BTC爆上$75K → Call輸$15K，淨輸$11K = 輸突接近3倍。

> **而家建議嘅 Long Strangle 屬於 🔴 Long（買方）陣營** — IV Rank 30-45 唔算貴、有 CPI + FOMC 催化劑、方向不明確。

## 八、總結：雙開策略的做與不做速查表

### ✅ 可以做雙開（Long Straddle/Strangle）的情況

| # | 情況 | 關鍵條件 |
|---|------|---------|
| 1 | **重大事件前**（FOMC、CPI、ETF、監管） | 事件日期明確，IV未過度膨脹 |
| 2 | **IV處於歷史低位** | IV Rank < 25, IV Percentile < 30% |
| 3 | **技術突破前夕** | 布林帶極窄、ATR低位、長期盤整 |
| 4 | **市場對波動定價不足** | IV低於預期實際波動 |
| 5 | **有足夠到期時間緩衝** | DTE > 事件日 + 7-14天 |
| 6 | **資金管理到位** | 單筆風險 ≤ 總資金 2-5% |

### ❌ 不可以做雙開（Long Straddle/Strangle）的情況

| # | 情況 | 關鍵指標 |
|---|------|---------|
| 1 | **IV極度偏高** | IV Rank > 90, IV Percentile > 95 |
| 2 | **事件後追單（IV Crush高風險）** | 事件已發生，IV仍高 |
| 3 | **無催化劑的低波動盤整** | ATR低位、布林帶無突破信號 |
| 4 | **流動性不足的合約** | OI < 50 BTC, Spread > 3% |
| 5 | **到期日過短（<7天）** | Theta呈指數級加速衰減 |
| 6 | **無法承受100%虧損** | 資金管理不匹配 |
| 7 | **完全可預測事件且市場已充分定價** | 如減半，IV早已膨脹 |
| 8 | **FOMO追單（已大幅波動後）** | 貪婪驅動，非策略驅動 |

### 🎯 核心心法

> **Long Straddle 的盈利公式 = 實際波動幅度 > 隱含波動率定價的幅度**
> 關鍵不在於「會不會波動」，而在於「市場是否**低估**了波動」。

> **Short Straddle 的盈利公式 = 隱含波動率 > 實際波動率 + 安全邊際**
> 關鍵在於在 IV 極高時進場，並嚴控尾部風險。
